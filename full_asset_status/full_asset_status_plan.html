Data Sources — How to Get Each Value

📈 Technical (all assets — stocks & crypto)

ValueSourceHow to CalcWhat to Record
Price & % Binance · ccxt (free) Use daily candle open (00:00 UTC) as price — fixed reference, doesn't drift during the day. % = (today_open ÷ yesterday_open − 1) × 100. Python: df['open'].iloc[-1] and df['open'].iloc[-2] from fetch_ohlcv('1d', limit=250). Open price in USD · % change open-to-open
EMA 200 TradingView (built-in) · pandas_ta (auto) Apply EMA(200) on daily close. Python: ta.ema(close, 200).iloc[-1]. Is close above (🟢) or below (🔴) EMA200? EMA200 value · 🟢 above / 🔴 below
ADX (14) TradingView → ADX/DMI · pandas_ta (auto) ADX(14) on daily — read the ADX line only (not +DI/−DI). Python: ta.adx(H,L,C,14)['ADX_14'].iloc[-1]. >20 = trending 🟢 · ≤20 = ranging 🔴. ADX value · 🟢 >20 / 🔴 ≤20
RSI (14) TradingView · pandas_ta (auto) RSI(14) on daily close. Python: ta.rsi(close, 14).iloc[-1]. Thresholds: 🔵 <30 (oversold bounce zone) · 🟢 30-70 (healthy) · 🟡 70–80 (overbought) · 🔴 >80 (extreme — expect pullback). RSI value · 🔵/🟢/🟡/🔴 emoji
Action Zone
(EMA 12/26 cross)		 TradingView daily · pandas_ta (auto) Compute EMA(12) and EMA(26) on daily close. Find last index where sign of (EMA12 − EMA26) changed:
🟢 = EMA12 crossed above EMA26 (bullish) · 🔴 = EMA12 crossed below EMA26 (bearish).
Python: diff = ema12 - ema26; cross_idx = np.where(np.diff(np.sign(diff)))[0][-1] + 1. Cross price = close[cross_idx]. Days = today_idx − cross_idx.
Shortcut: MACD(12,26,9) — MACD line crossing zero = same signal.		 🟢/🔴 · Cross price · Cross date · Days since
Key Levels (S/R) TradingView daily · Python pivot scan (auto) Scan swing highs/lows with window=2 (candle higher/lower than 2 neighbours on each side). Nearest swing high above price = Resistance. Nearest swing low below = Support. Python: pivot_highs = close[(close.shift(1)<close)&(close.shift(-1)<close)]; R = pivot_highs[pivot_highs > price].min(). Update when price breaks through a level. Resistance price · Support price
BOS
(Break of Structure)		 Python pivot scan (auto) · TradingView (manual verify) Detect swing highs (SH) and lows (SL) with window=2. Check last 3 of each:
• SH[−1] > SH[−2] && SL[−1] > SL[−2] → HH/HL intact (uptrend)
• SH[−1] < SH[−2] → LH forming — first warning
• Daily close below last confirmed SL → BOS confirmed
Python: sh = close[(close==close.rolling(5,center=True).max())]; sl = close[(close==close.rolling(5,center=True).min())]. Count consecutive HHs; detect first LH or SL break.		 HH/HL intact · LH forming · BOS confirmed · N consecutive HHs

🤖 AI Read (Claude judgment — no single formula)

ValueSourceHow to AssessWhat to Record
Wave
(Elliott Wave)		 TradingView daily — visual count Inspect last 20–50 daily candles. Count 5-wave impulse (1–5) or 3-wave correction (A-B-C). Key references:
• Wave 2 retraces 50–61.8% of Wave 1 → buy zone
• Wave 3 extends 1.618× Wave 1 → momentum phase
• Wave 4 retraces ~38.2% of Wave 3 → add zone
• Wave 5 at 100–161.8% extension → approach exit
• A-B-C = corrective / bearish phase
No pure algo count — requires visual judgment on which pivots are wave pivots vs noise.		 Wave 3 push / W4 pullback / W5 warn / A-B-C correction / Unclear · Key level context
Pattern
(Chart Patterns)		 TradingView daily — visual + partial algo Script detects candidates (Python):
• Bull flag = sharp rally + tight consolidation + declining volume
• Ascending triangle = flat resistance + rising lows
• Wedge = converging trendlines, often with RSI divergence
Claude confirms: Is the breakout real? Does pattern fit wave context? Is it at resistance or mid-range?
Types to watch: bull/bear flag · ascending/descending triangle · rising/falling wedge · ascending channel.		 Pattern name · or "No clear pattern"

Realized Price CoinMetrics Community API (free) — derived Derived from MVRV: Realized Price = Current Price / MVRV. Since MVRV = Market Cap / Realized Cap = Price / Realized Price, we can rearrange. No separate API call needed — use CapMVRVCur from CoinMetrics + current open price. Context: price above Realized = market in profit; below = historically strong buy zone. Realized Price value (USD)

F&G Index alternative.me (free API) Market-wide index (0–100). Free API: https://api.alternative.me/fng/ → JSON with value + value_classification. Labels: 0–24 Extreme Fear · 25–49 Fear · 50 Neutral · 51–74 Greed · 75–100 Extreme Greed. Updated daily. Value · Sentiment label
Dominance % CoinGecko API (free)
coinmarketcap.com/charts		 Dom% = this coin's market cap ÷ total crypto market cap × 100. CoinGecko free: GET /api/v3/global → market_cap_percentage object. Compare to yesterday's value for ↑ or ↓ direction. Dom% · ↑ rising / ↓ falling
Net Unrealized P/L
(NUPL)		 lookintobitcoin.com (BTC, free)
CoinMetrics Community API (free calc)
glassnode.com (ETH/SOL)		 NUPL = (Market Cap − Realized Cap) ÷ Market Cap. CoinMetrics free calc: (CapMrktCurUSD - CapRealUSD) / CapMrktCurUSD. Values: <0 = capitulation · 0–0.5 = optimism · 0.5–0.75 = belief/denial · >0.75 = euphoria. Thresholds by asset — see Section 3. NUPL value (e.g. 0.41) · 🟢/🟡/🔴 label
Exchange Reserve
+ Flows		 CoinMetrics Community API (free) Three metrics in one call: SplyExNtv (reserve BTC), FlowInExNtv (daily inflow BTC), FlowOutExNtv (daily outflow BTC). Reserve trend = today vs yesterday SplyExNtv: ↓ bullish, ↑ bearish. Net flow = FlowInExNtv − FlowOutExNtv: negative = net outflow (coins leaving, bullish ↓). Format in post: Reserve 2.64M↓ -4.5K↓ 🟢 Reserve value (e.g. 2.64M BTC) + trend ↓/↑ · Net flow ± with arrow · 🟢/🔴
Timing: Crypto at 07:00 UTC+7 = midnight UTC = daily candle close. US stocks: 07:00 UTC+7 is after 4pm ET close — fine to use.
Onchain cadence: Full read once per week. Daily: note only if a metric's label changed since last week.

ConditionLabel
